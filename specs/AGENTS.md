# Spec Format

## Estrutura

```markdown
# [Feature Name] Specification

## Overview
Descrição breve do que será implementado.

## Requirements
Lista de requisitos funcionais.

---

## Research: [Technology/Reference]
Análise de ferramentas/bibliotecas existentes.

### Option A: [Name]
| Pros | Cons |
|------|------|
| ... | ... |

### Option B: [Name]
...

---

## Final Recommendation

### Choice: [Recommended Option] ✅
Justificativa breve.

---

## Implementation Plan
- [ ] Task 1
- [x] Task completed

### Dependencies
- `package` - descrição

---

## Answers
1. **Q1**: Answer
2. **Q2**: Answer
```

## Regras

- Usar tabelas para comparação de opções
- Incluir prós e contras de cada alternativa
- Marcar ✅ na escolha recomendada
- Manter checkmarks de progresso durante implementação
- Colocar `---` para separar seções
- Listar dependências explicitamente
